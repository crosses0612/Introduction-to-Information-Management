# 系統架構說明

# 功能

1. 提醒廠商訂單的交貨日  
2. 協助管理原料數量（確認接單扣庫、進出貨紀錄、低庫存提醒）  
3. 紀錄客戶行為(如下訂的量、下單頻率)以作決策參考，甚至預測客戶週期  
4. 提供客戶下單介面  
5. 訂單過多時，系統會提醒客戶等待時間會較長

# 介面

1. 下單部分:  
   1. 客戶須建立會員帳戶才能下單  
   2. 一個訂單包括多筆商品以及交貨日  
   3. 廠商收到訂單後確認完畢再開始製作，同時，客戶會收到廠商接單的通知  
   4. 一旦訂單被確認後，首頁始終有出現交貨日的提醒  
   5. 廠商應能夠自行新增/刪除商品、調整其使用的原料比例  
   6. 待處理訂單可由客戶或廠商取消；確認接單後依配方扣減原料庫存  
2. 統計數據:  
   1. 通過會員系統掌控客戶訂貨週期  
   2. 掌握暢銷商品-  
3. 使用載體: 網頁

# 實作方式與細節

### **Frontend(廠商前端和後台)**

* React 

### **Backend**

* FastAPI / nodejs等等

### **Database**

* Neon（PostgreSQL）

# 市場區隔

傳統ERP（Enterprise Resource Planning）系統費用高、學習時間長，無法為不同產業客製化，因此本專案具有讓廠商調整配方比例，讓廠商能夠順應自己配方的調整，使原料管理更加有序。
